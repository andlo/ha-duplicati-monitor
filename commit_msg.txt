Move Recent runs table to its own panel view, fix table width with card-mod

andlo pointed out something I'd missed: he'd already tried widening
the table via the visual editor earlier today, and the CARD got
wider but the TABLE inside it didn't - meaning the whole `sections`/
`grid_options` investigation was solving the wrong problem. Confirmed
via web search: Home Assistant's markdown card strips `style`
attributes from its own rendered content for security, so even a raw
`<table style="width:100%">` inside the markdown gets silently
stripped - multiple community reports of the exact same symptom.

Two-part fix:
1. card-mod (already installed) injects CSS from outside the
   sanitized content area, bypassing the strip - `table { width: 100%
   }` now actually takes effect.
2. Moved the table into its own dedicated `type: panel` view (a new
   "Recent runs" tab) instead of trying to make it span columns in the
   main view. A panel view forces its one card to fill the entire
   page - no column/grid-span math involved at all, unlike every
   `sections`-view attempt so far, all of which produced results that
   didn't match Home Assistant's own documented sizing rules on this
   real dashboard.

Kept the main "Duplicati" view on masonry (stable, unaffected).

Updated tests/integration/test_dashboard_template.py's
_find_markdown_card() helper to search across all views, not just
views[0], since the table moved to a second view.

No custom_components/ changes - dashboard-only, no new release.
